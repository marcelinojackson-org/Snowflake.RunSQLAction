# Snowflake.RunSQLAction v1.1.0

## Highlights
- **Mandatory SQL input**: the action now fails fast if neither the `sql` input nor `RUN_SQL_STATEMENT` is provided.
- **Expanded row cap**: `return-rows` still defaults to 100 but can now go up to **10 000**, with server-side `LIMIT` injection when possible.
- **CSV artifacts**: set `persist-results: true` (or `RUN_SQL_PERSIST_RESULTS=true`) to capture the full rowset as CSV plus a metadata JSON file (filenames append the Snowflake query-id suffix). Directories are recreated each run so old files don’t pile up.
- **Documentation refresh**: README now follows a guided “steps” style with both basic and advanced usage examples.
